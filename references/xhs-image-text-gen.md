# XHS 图文生成链路（选题 → 可发布素材包）

目标：输入一个选题或内容描述，输出完整的小红书图文素材包（封面图 + 正文卡片 + 标题 + 正文 + 话题），**不执行发布**。
使用 `scripts/render_xhs.py` 本地渲染图片，不依赖外部 API。

## 总览

```
输入选题/描述
    ↓
Step 1: 内容策划（标题 × 3 + 正文 + 互动问句 + 话题）
    ↓
Step 2: 撰写渲染用 Markdown（frontmatter 封面 + 正文内容）
    ↓
Step 3: render_xhs.py 本地渲染（封面 + 正文卡片）
    ↓
Step 4: 素材组装（输出完整素材包）
    ↓
交付用户（不发布）
```

## 1) 输入

- `topic`：选题描述（1-2 句话即可）
- `style`（可选）：风格偏好（对应主题名：sketch/playful-geometric/neo-brutalism/botanical/professional/retro/terminal/default）
- `tone`（可选）：语气偏好（默认使用 `persona.md` 中的人设语气）
- `cover_count`（可选）：需要生成的封面数量（默认 1 张）

## 2) Step 1：内容策划

基于 `persona.md` 人设和选题输入，产出以下内容：

### 标题（3 个备选）
- 至少 1 个 ≤20 字（小红书标题限制）
- 句式优先：争议/反问/立场/情绪冲突
- 参考句式池：`被___惊到了` / `别再___了` / `折腾___后我麻了` / `___的人都懂`

### 正文（1 版可发布）
- 结构：开头钩子（1-2句）→ 观点+证据（2-3段）→ 互动收尾
- 短句为主，换行清晰
- 尾部预留话题区（用 `#话题` 格式）

### 互动问句（1 句）
- 放正文末尾或评论区第一条

### 话题（5-8 个）
- 核心话题 2-3 个 + 长尾话题 3-5 个
- 格式：`#话题名`

## 3) Step 2：撰写渲染用 Markdown

**注意：此 Markdown 专为图片渲染设计，禁止直接使用上一步的笔记正文。**

文档结构：

```markdown
---
emoji: "🔥"
title: "封面大标题（≤15字）"
subtitle: "封面副标题（≤15字）"
---

# 正文内容第一段

要点一，使用短句，少量 emoji 点缀...

要点二...

---

# 第二张卡片内容

...
```

### 封面 frontmatter 设计规则
- `emoji`：1 个代表性表情
- `title`：封面主文案，≤15 字，核心观点或情绪钩子
- `subtitle`：封面副文案，≤15 字，补充说明或反转

### 正文卡片设计规则
- 每张卡片内容不宜过长（auto-split 模式会自动处理）
- 若需手动控制分页，用 `---` 分隔，配合 `-m separator` 模式
- 适量使用 emoji 点缀（每段 1-2 个）
- 保持短句短段风格

## 4) Step 3：本地渲染图片

调用 `scripts/render_xhs.py`：

```bash
python scripts/render_xhs.py content.md \
    -t sketch \
    -m auto-split \
    --output-dir pic/
```

### 主题选择指南

| 主题 | 风格 | 适用场景 |
|------|------|----------|
| `sketch` | 手绘素描 | **默认推荐**，大字报/观点类 |
| `playful-geometric` | 活泼几何 | 轻松/趣味话题 |
| `neo-brutalism` | 粗野主义 | 冲击力强/态度鲜明 |
| `botanical` | 植物/清新 | 生活/自然/养生 |
| `professional` | 专业商务 | 职场/知识/教程 |
| `retro` | 复古怀旧 | 回忆/文化/情怀 |
| `terminal` | 极客终端 | 科技/编程/数码 |
| `default` | 简约灰 | 通用/无特殊要求 |

### 分页模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| `auto-split` | 自动按内容高度拆分 | **默认推荐**，内容长短不定 |
| `separator` | 按 `---` 手动分隔 | 需精确控制分页 |
| `auto-fit` | 固定尺寸缩放 | 内容确保一页放下 |
| `dynamic` | 动态高度 | 长内容不分页 |

### 渲染参数

- 默认尺寸：1080×1440px（3:4，小红书推荐比例）
- 完整参数文档：`references/xhs-render-params.md`
- 主题样式预览：`references/xhs-card-styles.md`

**生成后**：
- 图片自动保存到指定 `--output-dir` 目录（默认当前目录）
- 封面：`cover.png`
- 正文卡片：`card_1.png`、`card_2.png`...

## 5) Step 4：素材组装与草稿保存

素材生成完毕后，执行两个动作：**展示给用户** + **保存草稿文件**。

### 5.1 展示给用户（对话中输出）

```
📦 小红书图文素材包
━━━━━━━━━━━━━━━━━━━━

📸 封面图 + 正文卡片
- pic/cover.png（封面）
- pic/card_1.png ~ card_N.png（正文卡片）
- 主题：sketch | 模式：auto-split

📝 标题（3 选 1）
1. [标题A]（XX字）
2. [标题B]（XX字）✅ 推荐
3. [标题C]（XX字）

✍️ 正文
[完整正文内容]

💬 互动问句
[问句内容]

🏷️ 话题
#话题1 #话题2 #话题3 #话题4 #话题5

⚠️ 风险标注
- [是否涉及敏感话题/版权/夸大承诺]

━━━━━━━━━━━━━━━━━━━━
素材已生成完毕，未执行发布。
如需发布，请说"帮我发布这篇笔记"。
```

### 5.2 自动保存草稿文件（持久化）

每次生成素材包后，自动将完整素材写入 `knowledge-base/drafts/` 目录，确保跨会话可追溯。

**文件命名**：`YYYY-MM-DD-<slug>.md`（slug 从选题中提取 2-4 个关键词）

**草稿文件格式**：

```markdown
---
status: draft
created: YYYY-MM-DD HH:MM
topic: [选题描述]
render_theme: sketch
render_mode: auto-split
---

# [推荐标题]

## 封面图 + 正文卡片
- pic/cover.png（封面）
- pic/card_1.png
- pic/card_2.png

## 标题（3 选 1）
1. [标题A] ✅
2. [标题B]
3. [标题C]

## 正文

[完整正文内容]

## 互动问句

[问句内容]

## 话题

#话题1 #话题2 #话题3 #话题4 #话题5

## 风险标注

- [风险说明]
```

**状态值**：
- `draft`：已生成，待发布
- `published`：已发布（发布完成后更新）
- `archived`：已归档（不再发布）

**发布衔接**：用户说「帮我发布这篇笔记」或「发布最近的草稿」时：
1. 读取 `knowledge-base/drafts/` 下最新的 `status: draft` 文件
2. 从文件中提取封面路径、标题、正文、话题
3. 执行发布流程（`references/xhs-publish-flows.md`）
4. 发布成功后将 `status` 更新为 `published`

## 6) 常见触发语句

以下用户输入应触发本流程：
- "帮我生成一篇小红书图文"
- "帮我做一套小红书内容"
- "根据这个选题帮我出一套素材"
- "生成封面图和文案"
- "帮我写一篇小红书笔记（不发布）"
- "出一套关于 [主题] 的图文"

## 7) 与其他流程的衔接

- **选题灵感（3.2）→ 图文生成（4.5）**：选题产出后可直接进入图文生成
- **图文生成（4.5）→ 草稿保存**：自动写入 `knowledge-base/drafts/`，跨会话可追溯
- **草稿 → 发布（5）**：用户确认素材后，从草稿文件读取完整素材，衔接发布流程
- **Viral Copy（3.6）→ 图文生成（4.5）**：复刻分析后可用本流程生成新素材
- **知识库（6.5）**：生成结果可沉淀到 `knowledge-base/actions/`
